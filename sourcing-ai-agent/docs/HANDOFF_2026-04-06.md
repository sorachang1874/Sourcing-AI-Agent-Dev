## Product/Infra Direction Confirmed

后续这个项目的目标已经明确，不是“本地临时脚本集合”，而是：

- 在服务器上长期运行
- provider secrets 由 secret manager 注入
- runtime 高价值资产存到云端 object storage
- 重复或相近 query 复用既有 company/profile/evidence 资产
- 新 query 主要重新做：
  - plan
  - filtering
  - rerank
  - presentation

也就是说，后续系统更接近：

`asset acquisition platform + retrieval/presentation layer`

而不是每次都从零抓取所有数据。

## On Conversation Logs

没有必要刻意保存完整聊天原文，原因是：

- 原始对话太长，不利于新的 AI session 快速提取关键约束
- 结构化 handoff、progress、retrospective 更适合继续开发

真正应该保存的是：

- handoff summary
- progress
- retrospective
- exported asset bundles
- provider/bootstrap checklist

这份文档就是为此准备的。
